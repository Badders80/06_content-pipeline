Where the timings land

Durations are plotted in milliseconds. For a cinematic video (not interactive UI), the "sluggish" zone is less of a sin — but it still needs to be earned. The linear jockey zoom spans the full 10s scene without easing, which reads as mechanical rather than deliberate.

What's off

Issues to address
CriticalJakub

Hero jockey cutout zooms linearly for 10 seconds

WhatScene 4's transparent PNG cutout scales from 1.02 to 1.0 across the entire 300-frame scene using a plain interpolate() with default linear easing. There is no spring settle, no custom Bézier, no acceleration or deceleration.

Why it mattersThis is the hero shot — the subject is maximised and layered in front of the type. A constant-speed zoom feels robotic and cheapens the broadcast effect the layout is trying to achieve. The eye notices the motion because it never changes speed.

Recommended motionDrive the zoom with a spring (damping ~22, stiffness ~60, mass ~1) so it settles gently in the first 40–60 frames, then holds for the rest of the scene. The movement becomes a slow push-in that breathes, not a machine slide.

Text overlays fade in without blur

WhatTextOverlay animates translateY with a spring and opacity with a linear 30-frame ramp. There is no filter: blur() on enter or exit, so text materialises rather than coming into focus.

Why it mattersJakub's standard enter recipe is opacity + translateY + blur. Blur signals "coming into focus" and makes text entrances feel physical rather than like a CSS opacity toggle. On a 1080×1920 canvas where type is large, the difference is visible.

Recommended motionAdd a 4–6px blur that clears over the same 30-frame fade. Keep the spring for Y; pair it with the blur clearing so the text sharpens as it lands.

Issues to address
ImportantEmil

SlideWipe spring takes too long to settle

WhatSlideWipe uses a spring with damping: 20, stiffness: 120, mass: 0.8. That config needs roughly 35 frames to settle — about 1.17 seconds of a scene transition in a 40-second video.

Why it mattersEmil's rule is that UI motion should stay under 300ms; video transitions can be longer, but a 1-second+ wipe still costs narrative momentum. The wipe also uses a spring for a geometric clip-path reveal, which can feel soft where a crisp ease-out would feel more decisive.

Recommended motionTighten the spring (damping: 24, stiffness: 180, mass: 0.6) or switch to a 20-frame cubic-bezier(0.22, 1, 0.36, 1) ease-out. The reveal should feel like a clean cut, not a curtain rising.
